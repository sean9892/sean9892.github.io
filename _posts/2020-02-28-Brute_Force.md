모든 문제에 있어서 생각할 수 있는 가장 간단한 해법이 뭘까요?

수학적 지식이나 아이디어를 전혀(또는 거의) 사용하지 않고 가능한 모든 경우의 수를 살펴보는 것입니다.

이를 컴퓨터공학에서는 **완전탐색**, 또는 **Brute-Force(BF)**라고 부릅니다.

------

### How can we use BF?

완전탐색은 간단하지만 상당히 강력한 도구입니다.

가능한 모든 경우를 살펴보기 때문에 틀릴 일이 없다는 것이 장점이죠.

너무나 간단하고 직관적인 로직이고, 시간이 최대로 들어가기 때문에 대회에서 BF가 정해인 문제는 거의 나오지 않습니다.

하지만 수열의 n번째 항을 구하는 문제에서 초기항을 계산해서 규칙을 살펴볼 때 등의 특수한 경우나 제한이 작은 경우 BF를 사용해볼 수 있습니다.

여담으로, 예전에 존재했던 가장 간단한 암호 크래킹 방법 또한 BF였습니다. 지금은 시도횟수 제한 등으로 철저히 막혀있지만요.

[BOJ1018 체스판 다시 칠하기][https://www.acmicpc.net/problem/1018]

NxM의 보드에서 8x8의 보드판을 잘라내는데, 다시 칠해야하는 칸의 수를 최소화하는 문제입니다.

prefix sum 등으로 시간복잡도를 줄일 수 있습니다만, BF로 작성하여도 충분히 AC를 받을 수 있는 문제입니다.

우선 잘라내고자 하는 8x8의 보드판의 위치는 좌상단의 한 점만으로도 표현이 가능합니다.

이 때 가능한 좌상단 점의 수는 8x8 보드가 NxM 보드를 벗어나면 안 되므로 **(N-7)*(M-7)**개가 됩니다.

이를 2중반복문으로 모두 탐색한다고 생각하면, 2중반복문 내에서는 특정한 한 8x8 보드판에 대해 다시 칠해야하는 최소의 칸 수를 계산하면 됩니다.

좌상단 점이 흰색인 경우와 검은색인 경우를 각각 생각하여 일치하지 않는 칸의 수를 세고, 전체 최솟값에 이를 반영해주면 전체 최솟값을 구할 수 있습니다.

```cpp
#include<bits/stdc++.h>
using namespace std;
int n, m;
int arr[51][51];

int f(int x, int y, int color) {
	int res = 0;
	for (int i = x; i < x + 8; i++) {
		for (int j = y; j < y + 8; j++) {
			if (j % 2 == 0 && arr[i][j] != color) res++;
			else if (j % 2 == 1 && arr[i][j] == color) res++;
		}
		color = !color;
	}
	return res;
}

int main(void) {
	scanf("%d %d", &n, &m);
	for (int i = 0; i < n; i++) {
		for (int j = 0; j < m; j++) {
			char num;
			scanf(" %c", &num);
			if (num == 'W') arr[i][j] = 0;
			else arr[i][j] = 1;
		}
	}
	int ret = 9999;
	for (int i = 0; i <= n - 8; i++) {
		for (int j = 0; j <= m - 8; j++) {
			ret = min(ret, f(i, j, 0));
			ret = min(ret, f(i, j, 1));
		}
	}
	printf("%d\n", ret);
	return 0;
}
```

2309:

가장 간단하게는 고를 난쟁이가 7명이니 7중반복문을 구상할 수 있습니다.

하지만 이렇게는 N이 아무리 작아도 TLE를 피하지 못할 듯 하네요.

반대로, 9명 중 제외할 두 난쟁이를 고르는 것으로 생각해봅시다.

두 난쟁이를 고른다면 나머지 일곱 난쟁이를 선택한 것과 같습니다.

그 때 일곱 난쟁이의 키의 합은 미리 구해둔 아홉 난쟁이의 키의 합에서 선택된 두 난쟁이의 키를 뺀 것과 같습니다.

2231:

숫자 x를 문자열로 바꾸면, 그 길이는 (log(x)보다 작거나 같은 최대의 정수) + 1이 됩니다.

T의 분해합은 T+(T의 자릿수총합)이므로 T보다 큽니다.

따라서 N 이하의 모든 자연수의 생성자는 N 이하입니다.

분해합을 계산하는 과정은 길이에 비례, 즉 O(log(x))에 계산 가능하고

이것을 N개의 수에 대해 반복한다면 대략 O(NlogN)의 시간복잡도에 해결이 가능합니다.
