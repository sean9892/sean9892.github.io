ps를 하거나 무언가를 개발할 때에는 특정한 알고리즘을 사용하게 됩니다. 사소하고 단순한 것부터, 복잡하고 많은 구현량을 지닌 것까지 다양하게 사용되죠.

이 때, 보다 빠르고 원활한 서비스 제공, 혹은 솔루션 제작을 위해서는 여러 알고리즘의 성능을 비교할 필요가 있습니다.

알고리즘의 성능비교는 어떻게 하는 것일까요?

------

### What is Time Complexity?

알고리즘의 성능을 비교할 때에는 **시간복잡도(Time Complexity)**라는 개념을 사용합니다.

이는 어떠한 알고리즘이 주어진 입력과 동작에 소모되는 시간의 관계를 의미하는데, 이 관계를 좀 더 직관적이고 간단하게 볼 수 있도록 **빅오 표기법(Big-O notation)**을 주로 사용합니다.

(빅쎄타, 빅오메가, 스몰오 표기법 등 다양한 표기법이 있지만 이론 수업의 시험을 제외하면 본 기억이 없네요.)

```cpp
#include<stdio.h>

int main(void){
    int n,sum=0;
    scanf("%d",&n);
    for(int i=1;i<=n;i++){
        sum+=i;
    }
    printf("%d",sum);
}
```

위와 같이 1부터 n까지 더하는 프로그램을 작성했다고 하면, 이 프로그램은 n번의 더하기 연산을 실행합니다.

즉, 이 프로그램의 실행 시간은 n에 비례하는 셈이죠. 이를 빅오 표기법에서는 **O(n)**에 해당합니다.

```cpp
#include<stdio.h>

int main(void){
    int n;
    scanf("%d",&n);
    printf("%d",n*(n+1)/2);
}
```

그럼, 위와 같이 1부터 n까지의 합을 구하는 프로그램의 시간복잡도는 어떻게 될까요?

이 프로그램은 입력 n에 관계 없이 고정된 횟수의 연산을 실행합니다. 이 때는 프로그램의 시간복잡도가 1에 비례한다고 할 수 있으므로 **O(1)**에 해당합니다.

------

## What is Big-O notation?

빅오 표기법이란 정확히 뭘까요? 자세히 알아봅시다.

어떤 알고리즘의 연산 횟수가 *f(n)*회일 때, *O(g(n))*에 해당한다는 것은 다음과 같은 조건을 모두 만족한다는 뜻입니다.



적당한 양수 C와 a에 대해, C<k를 만족하는 모든 실수 k가

**f(k)<ag(k)**

를 만족한다.



예를 들어 f(n)=3n^2+6n+5일 때, g(n)=n^2이라면

*f(n)<ag(n)*을 만족하는 적당한 양수(ex. C=10000,a=10000)가 존재하므로 f(n)은 O(n^2)에 해당합니다.

상수를 적당히 잡다보면 세세한 계수보다는 식 자체의 차수가 중요합니다.

f(n)=3n^2+6n+5일 때 g(n)=n^5라면 이때 또한 적당한 양수 C, a가 존재하므로 f(n)은 O(n^5)에도 해당하지만, g(n)=n이라면 C, a가 어떤 값이 되도 조건을 만족할 수 없으므로 f(n)은 O(n)에는 해당하지 않습니다.



ps에서 자주 등장하는 시간복잡도는 다음과 같습니다.

1. **O(n)**
2. **O(n^2)**
3. **O(n^3)**
4. **O(nlogn)**

다음과 같은 이미지로 **O(n)**, **O(logn)**, **O(n^2)**, **O(n^3)**, **O(n!)**를 비교해 볼 수 있습니다.
